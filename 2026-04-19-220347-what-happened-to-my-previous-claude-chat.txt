⏺ Honest answer: no, it's free but not unlimited. Streamlit Community Cloud's
  free tier generally:

  - Caps you to a small number of public apps per account
  - Limits resources (memory/CPU) per app — fine for small dashboards like this

  - Sleeps apps after inactivity — first visitor after a quiet period waits

  - Requires the repo to be public (private repos usually need a paid plan)

  For a student project dashboard with a CSV and a few graphs, it comfortably
  stays within free limits and will work "indefinitely" as long as you don't
  delete it. But Streamlit changes their terms periodically, so check
  streamlit.io/cloud for the current limits before counting on it long-term.

  Hugging Face Spaces has similar free tiers with the same

⏺ Good questions.

  What's a web app? Software you use through a browser instead of installing it.
   Under the hood it's two pieces:
  1. Backend — Python running somewhere on a server, doing the actual work
  (loading your CSV, running k-means, etc.)
  2. Frontend — the webpage your browser renders (buttons, charts, tables)

  When you click something, the browser sends a request to the server, Python
  runs, and results come back. That's all Streamlit is doing — hiding the
  plumbing so you write one Python file.

  The "dashboard" you're remembering is probably one of these simpler options
  that don't need a running server:

  ┌─────────────┬────────────────────────┬──────────────────────────────────┐
  │   Option    │       What it is       │             Tradeoff             │
  ├─────────────┼────────────────────────┼──────────────────────────────────┤
  │             │ Free static hosting.   │ No Python runs. So no file       │
  │ GitHub      │ You publish            │ upload, no live k-means. Your 5  │
  │ Pages       │ HTML/images directly   │ graphs + text would just be a    │
  │             │ from your repo.        │ pretty webpage.                  │
  ├─────────────┼────────────────────────┼──────────────────────────────────┤
  │             │ Run a notebook, export │ Viewers see your analysis like a │
  │ Jupyter →   │  to a single .html     │  frozen report. No               │
  │ HTML export │ file, host on GitHub   │ interactivity.                   │
  │             │ Pages.                 │                                  │
  ├─────────────┼────────────────────────┼──────────────────────────────────┤
  │ Plotly      │ Interactive hover/zoom │ Charts are interactive, but      │
  │ charts in   │  charts, but data is   │ users can't upload new data or   │
  │ static HTML │ baked in.              │ rerun analysis.                  │
  └─────────────┴────────────────────────┴──────────────────────────────────┘

  So the tradeoff is: interactive = needs a server (sleeps on free tier). Static
   = free forever, no server, but frozen.

  For a class project showcase, a static GitHub Pages dashboard might actually
  be what you want — no sleep delays, always free, looks professional. Want me
  to switch to that instead of Streamlit?

  1. Go to https://github.com/GitLostintheSauce/Rock_Paper_Scissors in your
  browser.
  2. Click the Settings tab (top right of the repo page, next to "Insights").
  3. In the left sidebar, scroll down and click Pages.
  4. Under "Build and deployment":
    - Source: select "Deploy from a branch"
    - Branch: pick main from the first dropdown
    - Folder: pick / (root) from the second dropdown
    - Click Save
  5. Wait ~1-2 minutes. Refresh the Pages settings page — you'll see a green
  banner saying "Your site is live at 
  https://gitlostinthesauce.github.io/Rock_Paper_Scissors/".
  6. Click that URL — your dashboard is now public.

  If you don't see the Settings tab, you're probably not logged into the GitHub
  account that owns the repo (GitLostintheSauce). Sign in first.

  After it's live, any future git push auto-redeploys in ~30 seconds.
